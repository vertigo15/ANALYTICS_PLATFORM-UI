# Bug Fixes - February 21, 2026

## Issues Identified

### 1. Frontend JavaScript Error ❌
**Error:**
```
TypeError: Cannot read properties of null (reading 'slice')
```

**Root Cause:**
- Chart calculations were using `models` and `topUsers` arrays before data loaded from API
- Arrays were `null`/`undefined` during initial render
- Calling `.map()`, `.slice()`, `.forEach()` on null caused crashes

**Files Affected:**
- `web/src/app/dashboard/cost/page.tsx`

### 2. API 500 Error ❌
**Error:**
```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
/api/v1/agents/list
```

**Root Cause:**
- SQL query trying to select column `name` which doesn't exist
- Correct column name is `agent_name` in `gold.dim_agents` table
- Database error: "column 'name' does not exist" (PostgreSQL error code 42703)

**Files Affected:**
- `api/src/routes/agents.ts`

## Fixes Applied

### Fix #1: Added Null Safety to Frontend Charts ✅

**Changed in: `web/src/app/dashboard/cost/page.tsx`**

Added null coalescing operators to all array operations:

```typescript
// BEFORE (caused crash):
models.map((m) => ...)
models.forEach((model) => ...)
topUsers.slice(0, 10)

// AFTER (safe):
(models || []).map((m) => ...)
(models || []).forEach((model) => ...)
(topUsers || []).slice(0, 10)
```

## Lessons Learned

### 1. Always Check for Null/Undefined in React
**Problem:** Direct array operations without null checks
**Solution:** Use `(array || [])` pattern for optional arrays

```typescript
// GOOD PATTERN:
const data = (apiResponse || []).map(...)

// BAD PATTERN:
const data = apiResponse.map(...) // Crashes if null
```

### 2. Verify Column Names Against Schema
**Problem:** Assumed column name without checking database
**Solution:** Always reference master context or schema documentation

### 3. Docker Requires Rebuild for API Changes
**Problem:** Restarting container doesn't pick up TypeScript changes
**Solution:** Must run `docker compose build` to recompile

```powershell
# Wrong: Just restart
docker compose restart api  # ❌ Uses old compiled code

# Right: Rebuild and restart
docker compose build api    # ✅ Compiles TypeScript
docker compose up -d api    # ✅ Uses new compiled code
```
